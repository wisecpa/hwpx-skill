## 디자인 원칙

### 핵심

1. **주제에 맞는 색상을 선택한다** — 파란색 디폴트 금지. "이 색상을 완전히 다른 문서에 넣어도 어울린다면, 충분히 구체적인 선택이 아닌 것"
2. **매 섹션마다 시각 요소가 있어야 한다** — 텍스트만 있는 섹션은 잊혀진다
3. **같은 레이아웃을 반복하지 않는다** — 표→목록→박스→인용문 순환
4. **60-30-10 법칙** — 주색 60%, 보조색 30%, 악센트 10%

### Before Starting — 문서 생성 전 결정

1. **주제에 맞는 팔레트 선택** (아래 10종 중 선택 또는 커스텀)
2. **문서 성격 파악** — 공문서(절제)/보고서(데이터 중심)/제안서(설득)/안내문(친근)
3. **시각 모티프 결정** — 하이라이트 박스? 아이콘 표? 컬러 구분선? 하나를 골라 반복
4. **표지 필요 여부** — 3페이지 이하면 표지 불필요

## 주제별 색상 팔레트

문서 주제에 맞는 팔레트를 선택한다. Administrative Slate가 기본이지만, 주제가 명확하면 다른 팔레트를 사용.

| 테마 | Primary | Secondary | Accent | Surface | 용도 |
|------|---------|-----------|--------|---------|------|

| **Forest & Growth** | `#2C5F2D` | `#97BC62` | `#F5F5F5` | `#F9FBF7` | 환경, ESG, 성장 전략 |
| **Warm Executive** | `#B85042` | `#E7E8D1` | `#A7BEAE` | `#FBF9F7` | 제안서, 브랜딩 |

| **Coral Energy** | `#F96167` | `#F9E795` | `#2F3C7E` | `#FFFDF7` | 마케팅, 캠페인 |
| **Charcoal Minimal** | `#36454F` | `#F2F2F2` | `#212121` | `#FFFFFF` | 미니멀, 기술 문서 |

### 팔레트 적용 방법

```python
from pyhwpxlib import HwpxBuilder

# 팔레트 정의 (문서 시작 전 결정)
PALETTE = {
    'primary': '#2C5F2D',       # Forest — 헤더, 섹션 제목
    'on_primary': '#FFFFFF',    # primary 위 텍스트
    'secondary': '#97BC62',     # 보조 — 하이라이트 박스
    'accent': '#F5F5F5',        # 악센트 — 콜아웃
    'surface': '#F9FBF7',       # 배경
    'on_surface': '#2b3437',    # 본문 텍스트
    'on_surface_var': '#586064',# 메타데이터
}

doc = HwpxBuilder(table_preset='corporate')

# 섹션 제목 — primary 색상
doc.add_paragraph('1. 서론', bold=True, font_size=14, text_color=PALETTE['primary'])

# 하이라이트 박스 — secondary 배경
doc.add_table([['핵심 요약']],
    cell_colors={(0,0): PALETTE['secondary']}, header_bg='',
    cell_styles={(0,0): {'text_color': PALETTE['primary']}},
    row_heights=[2400], cell_margin=(400,400,300,300), use_preset=False)

# 표 헤더 — primary 배경
doc.add_table([['항목', '값'], ['A', '100']],
    header_bg=PALETTE['primary'])
```

## 레이아웃 패턴

### 섹션별 시각 요소 매핑

| 내용 유형 | 레이아웃 패턴 | HwpxBuilder 구현 |
|----------|-------------|-----------------|
| **핵심 수치** | 큰 숫자(24pt) + 작은 설명(10pt) | `add_paragraph(수치, font_size=24, bold=True)` + `add_paragraph(설명, font_size=10)` |
| **요약/결론** | 하이라이트 박스 (배경색) | `add_table([[텍스트]], cell_colors=...)` |
| **비교 데이터** | 2열 표 + 색상 대비 | `add_table(..., col_widths=[21260, 21260])` |
| **순서/프로세스** | 번호 목록 + 들여쓰기 | `add_numbered_list(items)` |
| **인용/참고** | 콜아웃 박스 (보라/파스텔) | `add_table([[텍스트]], cell_colors={(0,0): tertiary})` |
| **주의/경고** | accent 색상 텍스트 | `add_paragraph(텍스트, text_color=accent, bold=True)` |
| **타임라인** | 번호 + 날짜 강조 | `add_numbered_list` + 날짜는 bold |
| **인물/조직** | 키-값 표 (라벨 볼드) | `add_table(..., header_bg='', cell_styles={...})` |
| **장문 본문** | 2칸 들여쓰기 + 소제목 | `add_paragraph('  텍스트...', font_size=11)` |

### 섹션 구성 순환 패턴

같은 패턴이 연속하지 않도록 순환:

```
섹션1: 제목 + 하이라이트 박스 + 본문
섹션2: 제목 + 표 (데이터)
섹션3: 제목 + 본문 + 글머리표 목록
섹션4: 제목 + 콜아웃 박스 + 본문
섹션5: 제목 + 큰 수치 + 설명
```

### 대비 구조 (Sandwich)

긴 문서에서 시각적 리듬:
```
표지 — 어두운 배경 (primary 색상)
본문 — 밝은 배경 (surface)
결론 — 어두운 배경 또는 하이라이트 박스
```

## 타이포그래피

### 폰트 체계

| 요소 | 크기 | 스타일 | 색상 |
|------|------|--------|------|
| 문서 타이틀 | 18pt | bold, CENTER | primary |
| 섹션 제목 | 14pt | bold | primary |
| 본문 | 11pt | normal, 2칸 들여쓰기 | on_surface |
| 부제/캡션 | 10pt | normal | on_surface_var |
| 참고/미주 | 8pt | normal | outline_var |
| 큰 수치 | 24pt | bold | primary 또는 accent |
| 표 헤더 | 11pt | bold | on_primary (흰색) |

### 한글 폰트 선택

| 문서 유형 | 본문 | 제목 | 분위기 |
|----------|------|------|--------|
| 공문서/정부 | 바탕 | 바탕 Bold | 공식적, 전통적 |
| 기업 보고서 | 함초롬돋움 | 함초롬돋움 Bold | 현대적, 깔끔 |
| 학술 논문 | 바탕 | 바탕 Bold | 학술적, 진지 |
| 마케팅/제안 | 맑은 고딕 | 맑은 고딕 Bold | 세련, 친근 |

## 시각 요소 규칙

### 매 섹션 최소 1개

텍스트만 있는 섹션은 금지. 다음 중 하나를 반드시 포함:
- 표 (데이터/비교/키-값)
- 하이라이트/콜아웃 박스
- 글머리표/번호 목록
- 큰 수치 + 설명
- 구분선 (텍스트 방식)
- 이미지 (있는 경우)

### 색상 사용 비율

```
Primary:   60% — 헤더, 제목, 표 헤더 배경
Secondary: 30% — 하이라이트 박스, 정보 박스
Accent:    10% — 경고, 강조, 콜아웃
```

### 간격

| 요소 | 간격 |
|------|------|
| 제목 → 본문 | 빈 줄 없음 (바로 아래) |
| 섹션 → 섹션 | 빈 줄 1개 |
| 표 전후 | 빈 줄 1개씩 |
| 박스 전후 | 빈 줄 1개씩 |
| 문서 끝 주석 | 빈 줄 2개 |

## 피해야 할 실수 (Avoid)

**Assume there are problems. Your job is to find them.** 첫 번째 결과물은 거의 항상 문제가 있다.

### 레이아웃

- **같은 레이아웃 반복 금지** — 표→표→표 연속 금지. 표→목록→박스→인용문 순환
- **본문을 가운데 정렬하지 않는다** — 본문과 목록은 왼쪽/양쪽 정렬. 가운데는 제목만
- **폰트 크기 대비를 충분히** — 제목 18pt과 본문 11pt의 차이가 눈에 보여야 한다. 14pt 제목 + 12pt 본문은 구분이 안 됨
- **한 섹션만 꾸미고 나머지 방치 금지** — 전체적으로 일관되게, 아니면 아예 심플하게
- **텍스트만 있는 섹션 금지** — 매 섹션에 시각 요소 (표/박스/목록/수치) 최소 1개

### 색상

- **파란색 디폴트 금지** — 주제에 맞는 팔레트 선택 (위 10종 참조)
- **순검정(#000000) 사용 금지** — 반드시 on_surface(#2b3437 등) 사용
- **낮은 대비 금지** — 밝은 배경에 밝은 텍스트, 어두운 배경에 어두운 텍스트 금지
- **단조로운 파란색 일색 금지** — primary/secondary/tertiary/error 역할별 구분

### 구조

- **과도한 page_break 금지** — 자연스러운 흐름 우선. 주제 대전환에서만 사용
- **간격 랜덤 혼용 금지** — 빈 줄 0개 or 1개를 일관되게. 어떤 곳은 3줄, 어떤 곳은 0줄 금지
- **모든 문서에 같은 표지 금지** — 표지 필요 여부 먼저 판단. 3페이지 이하면 불필요
- **폰트 크기 점프 금지** — 18→14→11→10→8 점진적 축소 유지

### 내용

- **원문 요약/재작성 금지** — 원문 텍스트 그대로 사용. LLM은 배치와 스타일만 담당
- **"CONFIDENTIAL", "EQUITY RESEARCH" 같은 라벨 붙이기 금지** — 원문에 없으면 추가하지 않는다
- **모든 문서에 7개 섹션 + 목차 기계적 생성 금지** — 내용에 맞는 구조를 판단

### AI 생성 문서의 흔한 패턴 (피해야 함)

- **제목 아래 악센트 라인** — AI 생성 문서의 대표적 징후. 여백이나 배경색으로 대체
- **모든 섹션 동일한 구조** — 제목→본문→표 반복은 기계적으로 보인다
- **빈 줄 5개 → 카테고리 → 대제목 → 부제 → 박스 → 날짜** 패턴 — 매번 같은 표지

## QA 프로세스 (Required)

**문제가 있다고 가정하고 찾는다.** 0개 발견이면 충분히 안 본 것이다.

### Step 1: 구조 검증

```bash
pyhwpxlib validate output.hwpx
```

### Step 2: 내용 검증

```python
from pyhwpxlib.api import extract_text
text = extract_text("output.hwpx")
print(text)
```

체크:
- 누락된 내용 없는지
- 텍스트 순서가 맞는지
- 오타 없는지

### Step 3: 시각 검증 (사용자에게 요청)

```
"Whale에서 열어 확인해주세요. 수정할 부분이 있으면 알려주세요."
```

### Step 4: 시각 검증 체크리스트

```
□ 텍스트가 잘리거나 넘치지 않는가?
□ 표 열 너비가 내용에 맞는가? (긴 텍스트가 좁은 열에 들어가 있지 않은지)
□ 색상 대비가 충분한가? (밝은 배경에 밝은 텍스트 없는지)
□ 같은 레이아웃이 연속 반복되지 않는가?
□ 모든 섹션에 시각 요소가 있는가?
□ page_break가 적절한가? (내용이 잘리지 않는지)
□ 표 헤더와 데이터 행의 정렬이 일관적인가?
□ 글머리표/번호 목록의 들여쓰기가 본문과 구분되는가?
□ 하이라이트/콜아웃 박스의 패딩이 충분한가?
□ 원문 텍스트가 누락/변형 없이 보존되었는가?
```

### Step 5: 수정 → 재검증 루프

```
1. 생성 → validate → 내용 검증 → 사용자 시각 검증
2. 문제 발견 → 수정
3. 재검증 (수정이 다른 문제를 만들지 않았는지 확인)
4. 전체 패스에서 새로운 문제가 없을 때까지 반복
```

**한 번의 수정-검증 사이클을 완료하기 전에 성공을 선언하지 않는다.**
