## 문서 유형

| 유형 | 근거 | 용도 | 시행문 변환 |
|---|---|---|---|
| 일반기안문 | 규칙 제3조제1항, 별지 제1호서식 | 일반 발송 공문 | ✅ |
| 간이기안문 | 규칙 제3조제1항, 별지 제2호서식 | 내부결재 (보고서/계획서/검토서) | ❌ |
| 일괄기안 | 규칙 제5조제1항 | 관련 2+ 안건 동시 기안 | — |
| 공동기안 | 영 제8조제3항 | 2+ 기관 공동 명의 시행 | — |

## 구조

**일반기안문**:
- 두문: 행정기관명 + 수신 + (경유)
- 본문: 제목 + 내용 + 붙임
- 결문: 발신명의 + 기안자·검토자·결재권자 직위서명 + 시행/접수 + 주소/연락처/공개구분

**간이기안문**:
- 좌상 관리정보 (생산등록번호, 등록일, 결재일, 공개구분)
- 우상 결재란
- 중앙 제목 박스
- 하단 작성일 + 작성기관

## 일반기안문 생성

```python
from pyhwpxlib.gongmun import Gongmun, GongmunBuilder, signer

doc = Gongmun(
    # 두문
    기관명="행정안전부",
    수신="수신자 참조",           # 또는 "내부결재", "OOO장관(OO업무담당과장)"
    경유="",                      # 없으면 빈 문자열
    제목="2024년 정보공개 종합평가 계획 안내",

    # 본문
    관련문서="",                  # 선택: "OO부 OOO과-123(2025. 9. 20., '제목')호"
    본문=[
        "「공공기관의 정보공개에 관한 법률」제24조에 의거 ...",
        "평가대상 기관의 정보공개 담당자께서는 ...",
        "아울러, 평가계획 및 세부 평가지표에 대한 설명회는 ...",
    ],
    본문_자동번호=True,           # True면 1. 2. 3. 자동 부여
    붙임=["2024년 정보공개 종합평가 계획 1부."],  # 자동 "끝." 추가

    # 결문
    발신명의="행정안전부장관",
    수신자="2024년 정보공개 종합평가 대상 기관(554개)",
    기안자=signer("행정사무관", "김OO"),
    검토자=signer("서기관", "홍OO"),
    결재권자=signer("정보공개과장", "김OO",
                   전결=True, 서명일자="2025. 9. 30."),
    협조자=[],                    # Signer 목록
    시행_처리과명="정보공개과",
    시행_일련번호="000",
    시행일="2025. 9. 30.",
    우편번호="30112",
    도로명주소="세종특별자치시 도움6로 42 정부세종청사 중앙동",
    홈페이지="www.mois.go.kr",
    전화="(044)205-0000",
    팩스="(044)204-0000",
    이메일="abcde@mois.go.kr",
    공개구분="대국민공개",        # "공개" | "부분공개" | "비공개" | "대국민공개"
    관인생략=False,
    서명생략=False,
)
GongmunBuilder(doc).save("output.hwpx")
```

### signer 보조 함수

```python
from pyhwpxlib.gongmun import signer

# 기본
s = signer("행정사무관", "김갑동")

# 전결 (결재권 위임받아 서명)
s = signer("과장", "홍길동", 전결=True, 서명일자="2025. 9. 30.")

# 대결 (결재권자 대신 서명)
s = signer("과장", "박영희", 대결=True, 서명일자="2025. 9. 30.")

# 발의자 (★)
s = signer("사무관", "김X", 발의자=True)

# 보고자 (●) — 직접 결재권자에게 보고 시만
s = signer("주무관", "이X", 보고자=True)
```

## 간이기안문 생성

```python
from pyhwpxlib.gongmun import GongmunSimple, GongmunBuilder, signer

doc = GongmunSimple(
    # 관리정보
    생산등록번호="정보공개과-840",
    등록일="2025. 9. 20.",
    결재일="2025. 9. 20.",
    공개구분="대국민공개",

    # 결재란 (직위별)
    결재_서명자=[
        signer("주무관", "김OO"),
        signer("행정사무관", "이OO"),
        signer("정보공개과장", "박OO"),
        signer("정부혁신국장", "황OO"),
    ],
    협조자=[],

    # 제목/요약
    제목="행정업무운영 편람 발간 계획",
    보고근거_요약="필요한 경우 보고근거 및 보고내용을 요약하여 적을 수 있음",

    # 하단
    작성일="2025. 9. 20.",
    작성기관_부="행정안전부",
    작성기관_국과="정보공개과",
)
GongmunBuilder(doc).save("output.hwpx")
```

**중요**: 간이기안문은 시행문으로 변환 불가 (규칙 제3조제3항). 내부결재 전용.

## 규정 검증

```python
from pyhwpxlib.gongmun import validate_file, validate_text, format_report

# 파일 검증
findings = validate_file("output.hwpx")
print(format_report(findings))

# 텍스트 검증
findings = validate_text("수신 OOO장관\n제목 OOO\n... 참석 바람.")
for f in findings:
    print(f"[{f.severity}] {f.code}: {f.message}")
    print(f"  → '{f.excerpt}' ({f.rule_ref})")
```

## 편람 규칙 전체

기계판독용 YAML: [../pyhwpxlib/gongmun/rules.yaml](../../pyhwpxlib/gongmun/rules.yaml)

### 표기 규칙 요약
- **날짜**: `YYYY. M. D.` (마침표 + 공백, 0 제거) — 예: `2025. 9. 20.`
- **시간**: `HH:MM` (24시각제) — 예: `15:20`
- **금액**: `금113,560원(금일십일만삼천오백육십원)`
- **끝 표시**: 본문/붙임 끝에 2타(스페이스 2개) + `끝.`
- **2타**: 한글 1자 = 영숫자 2자 = 스페이스 2회

### 항목 기호 8단계
```
1. → 가. → 1) → 가) → (1) → (가) → ① → ㉮
```
이후: 하 → 거 → 너 → 더 ... (단모음 순)

### 들여쓰기
- Level n+1은 Level n에서 2타 들여쓰기
- 기호와 내용 사이: 1타
- 두 줄 이상: 둘째 줄부터 내용 첫 글자에 정렬

### 어투 (편람 §3 p47-48)
- ✅ "하시기 바랍니다", "하십시오", "드립니다"
- ❌ "할 것", "~바람", "~음" (위압감)
- ❌ "치하했다", "영접" (권위적)
- ❌ "결손가정", "학부형" (차별적)

### 개방형 문서 (편람 §3 p35)
- 권장: hwpx, odt, docx, pdf (복수 제공)
- 표 셀 병합·분리 지양
- 도형 대신 특수문자 (→, ⇒)
- 한글 호환 영역 특수문자 지양 (유니코드 표준 우선)

## 1페이지 자동 맞춤 (autofit, 0.12.0+)

공문은 "1건 1매" 원칙(편람 §4 p68)이 있지만, 본문이 약간 많으면 결문이 2페이지로 밀림. `autofit=True` 로 rhwp RenderTree 기반 자동 조정:

```python
GongmunBuilder(doc, autofit=True).save("output.hwpx")
```

**조정 순서** (누적, 최대 3회 재빌드):

| 단계 | 조작 | 1회당 | 하한 |
|:---:|------|------|------|
| 0 | spacer pt (6 → 4) | −2pt | 4pt |
| 1 | lineSpacing 전역 × 0.90 (본문 160 기준) | × 0.90 | 120 (작은 값은 post-save 클램프) |
| 2 | 상·하 여백 | −2mm | 12mm |

- **Fail-soft**: 수렴 실패 시 `WARN` 로그("compact=True 또는 본문 축소 검토") + 마지막 조정본 유지.
- **사용자 설정 존중**: `compact=False` 를 autofit이 뒤집지 않음. 필요하면 사용자가 명시적으로 `compact=True` 결합.
- **수렴 불가 케이스**: 8~9 항목×서브 4개 이상 — 본문 자체를 줄여야 함.
- **성능**: `autofit=False` 경로는 추가 비용 0 (rhwp import조차 안 함). True일 때만 WASM 싱글톤 로드.

**rhwp는 auto-paginator이므로** 페이지 내부 y+h 비교로는 overflow를 알 수 없다. 구현은 `page_count > 1`을 overflow 신호로 삼는다 ([design doc §5.1](../../docs/02-design/features/compact-autofit.design.md)).

## 보고서 vs 공문 차이

| 구분 | 보고서 | 공문 |
|---|---|---|
| 목적 | 내부 검토·발표 (시각 전달) | 공식 송수신 (법정 서식) |
| 폰트 | 신명 견고딕/태고딕 등 브랜드 폰트 | 굴림체/맑은 고딕 등 가독성 |
| 제목 | 22~28pt | 13pt (수신·경유·제목 동일) |
| 본문 | 15pt | 12pt (줄간격 160%) |
| 구조 | 표지 + TOC + 섹션 | 두문 + 본문 + 결문 |
| 사용 도구 | `HwpxBuilder(theme='forest')` | `GongmunBuilder(Gongmun(...))` |

**섞지 말 것**: 공문 폰트/크기/구조는 법정 표준이므로 보고서 스타일을 강제하지 않음.

## 트러블슈팅

**Q. 생성 HWPX가 Whale에서 안 열림**
→ `pyhwpxlib validate output.hwpx` 로 XML 검증. `validate_file`은 *내용* 검사만 함.

**Q. 특수문자 경고(㉮ 등)가 뜨는데 편람에서는 사용함**
→ 편람도 현 버전은 ㉮을 쓰지만, 전자 판독성을 위해 유니코드 표준(①~⑳ U+2460~) 사용 권장. INFO 레벨이라 무시 가능.

**Q. AI, MOU 등 영문 약어 INFO 경고**
→ 첫 등장 시 `AI(인공지능)` 형식으로 한글 설명 1회 붙이면 해소.

**Q. 발신명의 뒤 관인(검정 박스) 필요**
→ 현재 최소 구현에는 관인 이미지 삽입 기능 없음. `doc.관인생략=True` 로 "(관인생략)" 텍스트 대체. 실제 관인은 Whale에서 수동 첨부.
